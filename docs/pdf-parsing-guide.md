# PDF解析详解

## 火车票解析

### 支持格式

12306电子发票（铁路电子客票）PDF格式

### 提取字段

| 字段 | 提取方式 | 示例 |
|------|----------|------|
| 发票号码 | `发票号码[:\s]*(\d+)` | 26119110010002383960 |
| 出发站 | 英文站名顺序判断 | 北京南站 |
| 到达站 | 英文站名顺序判断 | 南京南站 |
| 车次 | `(G\d+|D\d+|C\d+)` | G11 |
| 日期 | 行程日期（非开票日期） | 2026年01月25日 |
| 时间 | `(\d{1,2}:\d{2})开` | 12:00开 |
| 座位 | `(\d+车\d+[A-Z]号)` | 14车13A号 |
| 价格 | `票价[:\s]*[￥¥Y]?\s*(\d+\.\d{2})` | ￥533.00 |

### 日期提取逻辑

火车票PDF包含两个日期：
- **行程日期**：实际乘车日期（需要提取）
- **开票日期**：发票开具日期（用于记录）

提取策略：
1. 查找日期后紧跟"开"时间的（行程日期）
2. 查找英文站名后的日期
3. 跳过"开票日期"字段的日期

### 站站识别逻辑

PDF中站名可能分散排列：
```
北京南    站
Beijingnan Nanjingnan
南京南    站
```

识别策略：
1. 首先匹配完整站名 `北京南站|南京南站`
2. 使用英文站名顺序判断出发/到达
   - `Beijingnan` 先出现 → 北京是出发站
   - `Nanjingnan` 先出现 → 南京是出发站

## 住宿发票解析

### 支持格式

增值税电子发票（普通发票）PDF格式，服务类型为"住宿服务"

### 提取字段

| 字段 | 提取方式 | 示例 |
|------|----------|------|
| 发票号码 | `发票号码[：:\s]*(\d{20})` | 26322000000821253661 |
| 开票日期 | `(\d{4})年(\d{1,2})月(\d{1,2})日` | 2026年01月30日 |
| 酒店名称 | `([一-龥]+酒店)` | 南京黄埔大酒店 |
| 天数 | `天\s*(\d+)` 或 `(\d+)\s*天` | 5 |
| 金额 | 金额列 | ¥1792.45 |
| 税额 | 税额列 | ¥107.55 |
| 合计 | `价税合计.*?(¥\d+\.\d{2})` | ¥1900.00 |

## 防重复计算

### 记录机制

系统维护 `processed_invoices.json` 文件：

```json
{
    "tickets": ["发票号码1", "发票号码2", ...],
    "invoices": ["发票号码1", ...]
}
```

### 处理流程

1. 加载已处理记录
2. 解析PDF时检查发票号码是否已处理
3. 跳过已处理的发票
4. 新发票处理后追加记录
5. 保存更新后的记录

### 清除记录

调用 `/api/clear` 接口或点击前端"清除记录"按钮。
