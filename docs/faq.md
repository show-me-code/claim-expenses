# 常见问题 FAQ

## Q1: 为什么处理后金额是0？

**可能原因及解决方案：**

| 原因 | 解决方案 |
|------|----------|
| 所有发票已被处理过 | 点击"清除记录"按钮，然后重新处理 |
| 票据没有形成闭环 | 检查是否有返程票，确保目的地匹配 |
| 文件夹路径不正确 | 确认路径格式正确（使用完整路径） |
| PDF格式不支持 | 检查PDF是否为支持的发票类型 |

## Q2: 什么是"闭环"？为什么不闭环不计入？

差旅报销要求行程完整：
- 有去程必须有返程
- 只有单程票无法证明行程结束
- 系统只计算形成闭环的行程

**闭环示例：**
- ✅ 北京→南京 + 南京→北京 = 闭环（计入）
- ❌ 北京→南京 = 不闭环（不计入）
- ✅ 上海→南京 + 南京→北京 = 闭环（休假期间出差）

## Q3: 伙食补助怎么计算？

```
伙食费 = 行程天数 × 每日标准

行程天数 = (返程日期 - 去程日期) + 1

示例：
去程：2026-01-25
返程：2026-01-30
天数：30-25+1 = 6天
补助：6 × 100 = ¥600
```

## Q4: 退票费怎么处理？

退票费发票会自动匹配到对应行程：
1. 系统识别退票费发票（含"退票费"字样）
2. 检查退票日期是否在行程区间内
3. 将退票费计入该行程的交通费小计

**注意：** 退票费发票需要与行程日期匹配才能计入。

## Q6: 为什么第二次处理显示"没有发现新的票据"？

这是防重复计算机制正常工作：
- 第一次处理已记录发票号码
- 第二次处理自动跳过已记录的发票
- 如需重新计算，点击"清除记录"

## Q9: 服务器端口被占用怎么办？

默认端口5000，如被占用：

方案一：停止占用进程
```bash
taskkill /F /IM python.exe
```

方案二：修改端口（app.py最后一行）
```python
app.run(debug=True, port=5001)  # 改为其他端口
```

## Q11: 为什么休假期间出差的闭环类型不同？

| 类型 | 说明 | 标签颜色 |
|------|------|----------|
| standard | 北京出发的标准出差 | 绿色 |
| vacation | 休假期间出差（去程非北京） | 橙色 |

两者计算方式相同，仅用于区分出差场景。

## Q12: 如何设置Base城市？

系统支持三种方式设置Base城市：

1. **下拉选择**：从15个常用城市中选择（北京、上海、广州等）
2. **自动检测**：点击"自动检测Base城市"按钮，系统分析票据后推荐
3. **默认自动**：不选择时，系统自动检测最频繁出发城市

**闭环匹配规则：**
- Base → 出差地 → Base（标准闭环）
- 休假地 → 出差地 → Base（休假期间出差）
