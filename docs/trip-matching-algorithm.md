# 闭环识别算法详解

## 闭环定义

差旅报销要求行程形成闭环：有去程必有返程。系统只计算闭环行程的费用。

## 支持的闭环模式

### 模式一：标准闭环（北京出发）

```
北京(Base) → 出差地 → 北京(Base)
```

**识别条件：**
- 去程票：出发站包含"北京"
- 返程票：到达站包含"北京"，出发站=去程的到达站
- 时间：返程日期 ≥ 去程日期

**示例：**
```
去程：北京南站 → 南京南站 (G11, 2026-01-25)
返程：南京南站 → 北京南站 (G744, 2026-01-30)
结果：闭环成功，计算费用
```

### 模式二：休假期间出差

```
休假地(不确定) → 出差地 → 北京(Base)
```

**识别条件：**
- 去程票：出发站不含"北京"，到达站=出差地
- 返程票：到达站包含"北京"，出发站=去程的到达站
- 时间：返程日期 ≥ 去程日期

**示例：**
```
去程：上海站 → 南京南站 (G1, 2026-01-26)
返程：南京南站 → 北京南站 (G2, 2026-01-28)
结果：闭环成功（休假期间出差），计算费用
```

## 不闭环的情况

以下情况不计入费用：

| 情况 | 说明 |
|------|------|
| 只有去程 | 缺返程票 |
| 只有返程 | 缺去程票 |
| 目的地不匹配 | 去程和返程城市不同 |
| 时间矛盾 | 返程早于去程 |

## 退票费处理

退票费发票会自动匹配到对应闭环：

```
条件：退票日期在 [去程日期, 返程日期] 区间内
结果：退票费计入该行程的交通费小计
```
